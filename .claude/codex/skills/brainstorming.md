# Brainstorming

Source workflow: `.claude/skills/brainstorming/SKILL.md`.

Use for `/brainstorming`, `brainstorming`, or `brainstorm <topic>` when work is
large, ambiguous, cross-cutting, or risky enough that success criteria must be
locked before planning.

## Codex Wrapper

1. Explore only the current repo surface relevant to the idea.
2. Ask focused questions for goal, success criteria, constraints, and
   non-goals.
3. Present 2-3 approaches with a recommendation.
4. Write the approved spec to `.claude/specs/YYYY-MM-DD-<slug>-spec.md`.
5. Recommend `/tailor` or `/writing-plans` as the next step.

Do not plan or implement until the user approves the spec.
