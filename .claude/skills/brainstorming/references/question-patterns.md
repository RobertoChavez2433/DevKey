# Question Patterns for Brainstorming

## Multiple Choice Templates

### Target Audience
"Who is the primary user for this feature?"
- Power users / developers
- General Android users
- Both equally

### Scope
"How much should we build in the first version?"
- Minimal viable (ship fast, iterate)
- Feature-complete (longer but polished)
- Progressive (phased releases)

### Technical Approach
"Which approach should we take?"
- Option A (recommended) — [benefits]
- Option B — [benefits]
- Option C — [benefits]

### Priority
"What matters most for this feature?"
- Performance
- Simplicity
- Flexibility / extensibility

## Open-Ended Templates

### Intent
"What problem does this solve for you?"

### Constraints
"Are there any hard requirements I should know about?"

### Success Criteria
"How will we know this feature is working well?"
