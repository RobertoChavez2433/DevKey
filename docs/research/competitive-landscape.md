# Competitive Landscape

**Date**: 2026-02-23

## DevKey Positioning

DevKey sits at the intersection of two markets that no existing keyboard serves well:
1. **Power-user keyboards** (terminal keys, modifier support) — served by Hacker's Keyboard (dormant)
2. **Modern consumer keyboards** (AI prediction, voice, themes) — served by SwiftKey, Gboard

## Key Competitors

### Gboard (Google)
- **Strengths**: Low resource usage, deep Google integration, 900+ languages, Emoji Kitchen
- **Weaknesses**: No true modifier keys, no Ctrl/Alt, limited customization, no arrow keys toggle
- **Relevance to DevKey**: Main competitor for general users. DevKey differentiates with power-user features + command-aware AI.

### Microsoft SwiftKey
- **Strengths**: Best swipe/flow, adaptive prediction, Copilot AI, cross-device clipboard
- **Weaknesses**: Resource-heavy, closed source, Microsoft account push
- **Relevance to DevKey**: Main UX inspiration. DevKey takes similar modern UX but adds terminal/developer features and stays open-source.

### Unexpected Keyboard
- **Strengths**: Copyleft, actively maintained, F-Droid, programmer-friendly
- **Weaknesses**: Limited prediction, no AI, basic theming
- **Relevance to DevKey**: Closest competitor in the power-user FOSS space. DevKey differentiates with AI prediction and modern UX.

### Termux:Float / Terminal-specific keyboards
- **Strengths**: Purpose-built for terminal
- **Weaknesses**: Not general-purpose keyboards
- **Relevance to DevKey**: DevKey replaces the need for a separate terminal keyboard by integrating command mode.

## DevKey's Unique Value Proposition

**"The keyboard that's as comfortable in a terminal as it is in a text message."**

1. **Command-aware autocorrect** — No other keyboard auto-detects terminal apps and switches to command prediction
2. **Full PC keyboard + modern UX** — No other keyboard combines 5-row layout + F-keys + Material You + AI prediction
3. **On-device AI, open source** — Privacy-first, no cloud dependency, fully auditable
4. **Smart hybrid mode** — Seamlessly transitions between text and command prediction based on context
